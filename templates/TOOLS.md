# TOOLS.md — Local Cheat Sheet

Environment-specific notes, aliases, and references. Separate from skills (which are shared) — this is your specific setup.

## Why separate

Skills are shared and reusable across users. Your setup notes are yours and shouldn't leak when you share skills.

## What belongs here

### Cameras / devices
- Camera names and locations
- Device nicknames
- Speaker / room names

### SSH / networking
- SSH hosts and aliases
- Static IPs if relevant
- Home network specifics

### Accounts / integrations
- Which account to use for which tool (don't put passwords — point to your secret manager)
- Preferred platforms for specific tasks

### TTS / voice
- Preferred voices
- Default speakers / rooms
- Voice personas per context

### Shortcuts
- Command aliases
- Path shortcuts
- Project-specific conventions

## Examples

```markdown
### Cameras
- living-room → Main area, 180° wide angle
- front-door → Entrance, motion-triggered

### SSH
- home-server → 192.168.1.100
- vps → via Tailscale hostname

### TTS
- Preferred voice: "Nova" (warm)
- Default speaker: Kitchen HomePod
```

*This is your cheat sheet. Keep it honest and up-to-date. When something changes (new device, new account, new alias), update this file so the agent doesn't keep using stale info.*
